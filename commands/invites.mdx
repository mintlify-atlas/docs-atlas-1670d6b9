## Overview

The `/invite_link` command generates shareable invite links for channels. Unlike `/invite` which requires specifying a username, invite links can be shared with anyone.

## /invite_link Command

Create an invite link for the current channel.

### Syntax

```bash
/invite_link
```

### Parameters

No parameters required.

Source: `src/lib/commands.ts:321`

### Restrictions

<ParamField path="privateOnly" type="boolean" default="true">

</ParamField>

<ParamField path="adminOnly" type="boolean" default="false">
  Available to all channel members (not admin-only)
</ParamField>

<ParamField path="channelOnly" type="boolean" default="true">
  Only available in channels, not DMs
</ParamField>

Source: `src/lib/commands.ts:318-320`

### Event Type

Triggers event: `create_invite_link`

Source: `src/lib/commands.ts:322`


The server determines the channel from the current context.

### Context-Based

The server uses the current channel context to:
1. Identify which channel to create the link for
2. Generate a unique invite token
3. Return the shareable URL

## Permission Model

Key difference from `/invite` and `/remove`:

| Command | Admin Only |
|---------|------------|
| `/invite` | Yes (`true`) |
| `/remove` | Yes (`true`) |
| `/invite_link` | No (`false`) |

**Any channel member** can create invite links, not just admins.

## Channel Context Requirement

The command requires channel context:

- Command fails in DMs
- Must be used within a channel
- Server determines which channel from context

## Private Context Requirement

The command requires private context:

```typescript

```

This likely means:
- Only works in private channels or admin views
- May not work in public channels
- Implementation detail depends on application's privacy model

## Error Scenarios

Potential error cases:

| Scenario | Expected Behavior |
|----------|------------------|
| Used in DM | Command rejected (channelOnly) |
| Used in public channel | May be rejected (privateOnly) |
| Server error | Link generation fails |
| No channel context | Command unavailable |

Note: The validation happens at the application level based on the `channelOnly` and `privateOnly` flags.
