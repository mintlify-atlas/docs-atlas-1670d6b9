## Overview

Channel management commands allow administrators to control channel membership. These commands require admin permissions and only work in channel contexts.

## /invite Command

Invite a user to join the current channel.

### Syntax

```bash
/invite username
```

### Parameters

<ParamField path="user" type="username" required>
  Username of the person to invite
  
  **Type:** `username`
  
  **Source:** `search` (async search results)
  
  **Prefix:** `` (empty string - no @ symbol required)
  
  **Validation:**
  - Must match a valid username from search results
  - Case-insensitive matching
  - Provides autocomplete suggestions as you type
</ParamField>

Source: `src/lib/commands.ts:291-301`

### Restrictions

<ParamField path="privateOnly" type="boolean" default="true">
  Only works in private/admin contexts
</ParamField>

<ParamField path="adminOnly" type="boolean" default="true">
  Requires admin permissions
</ParamField>

<ParamField path="channelOnly" type="boolean" default="true">
  Only available in channels, not DMs
</ParamField>

Source: `src/lib/commands.ts:294-296`

### Event Type

Triggers event: `invite_user`

### Payload

The command extracts the following data:

```typescript
{
  username: string,  // Matched username
  user_id: number    // User's ID from search results
}
```

Source: `src/lib/command-parser.ts:204-226`

## /remove Command

Remove a user from the current channel.

### Syntax

```bash
/remove username
```

### Parameters

<ParamField path="user" type="username" required>
  Username of the channel member to remove
  
  **Type:** `username`
  
  **Source:** `subscribed_without_self` (current channel members excluding you)
  
  **Prefix:** `` (empty string - no @ symbol required)
  
  **Validation:**
  - Must match a current channel member
  - Cannot remove yourself
  - Case-insensitive matching
  - Provides autocomplete of channel members
</ParamField>

Source: `src/lib/commands.ts:303-313`

### Restrictions

## Username Parameter Type

Both commands use the `username` parameter type with different sources:

### Source Types

<ParamField path="source" type="enum">
  Determines which user list to validate against:
  
  - **`all`** - All present users in the application
  - **`subscribed_without_self`** - Channel members excluding current user (used by `/remove`)
  - **`not_subscribed`** - Users not in the current channel
  - **`search`** - Async search results (used by `/invite`)
</ParamField>

Source: `src/lib/commands.ts:17` and `src/lib/commands.ts:126-138`

### Validation Flow

Username validation follows this process:

```typescript
// 1. Check prefix requirement
if (p.prefix && !rawValue.startsWith(p.prefix)) {
  return { isValid: false, error: `Must start with ${p.prefix}` };
}

// 2. Normalize value (remove prefix)
const username = normalizeUsernameValue(rawValue, p.prefix);

// 3. Get filtered user list based on source
const users = getUserList(); // Depends on source type

// 4. Find matching users for autocomplete
const matchingUsers = users.filter(u =>
  u.username.toLowerCase().startsWith(username.toLowerCase())
);

// 5. Check exact match for validation
const exactMatch = users.find(
  u => u.username.toLowerCase() === username.toLowerCase()
);

return {
  isValid: !!exactMatch,
  error: exactMatch ? undefined : "User not found",
  suggestions: matchingUsers.map(u => formatUsernameSuggestion(u.username, p.prefix))
};
```

Source: `src/lib/commands.ts:87-170`

## Autocomplete Behavior

### Dynamic Filtering

As you type, autocomplete filters suggestions:

```typescript
// For /invite (search source)
const matching = asyncSearchResults.filter(r =>
  r.username.toLowerCase().startsWith(input.toLowerCase())
);

// For /remove (subscribed_without_self source)
const subscribedWithoutSelf = subscribedUsers
  .filter(u => u.username !== currentUsername)
  .filter(u => u.username.toLowerCase().startsWith(input.toLowerCase()));
```

Source: `src/lib/commands.ts:111-113` and `src/lib/commands.ts:158-160`

### Suggestion Format

Suggestions include the username prefix:

```typescript
const formatUsernameSuggestion = (username: string, prefix: string) =>
  prefix ? `${prefix}${username}` : username;
```

Since both commands use empty prefix, suggestions are just usernames.

Source: `src/lib/commands.ts:76-77`
